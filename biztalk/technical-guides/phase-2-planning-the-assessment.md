## <a name="non-microsoft-software-considerations"></a>Microsoft 以外のソフトウェアの考慮事項  
 どの利害関係者は、パフォーマンスの評価時に使用される任意のサードパーティ製ソフトウェアの必要な専門知識を提供は、最初から非常に明確にすることをお勧めします。 Microsoft 以外のソフトウェアは、ソリューションで使用される場合は、次を検討します。  
  
-   ソフトウェアまたはハードウェアの必要のある判断を取得します。  
  
-   容量の計画し、する Microsoft 以外のソフトウェアを確実にサイズ変更が、ソリューション内のボトルネックになりません。  
  
-   必要な Microsoft 以外のソフトウェアをインストールするための行動計画を決定します。  
  
-   アクションを構成し、必要な Microsoft 以外のソフトウェアを最適化するための計画を作成します。  
  
## <a name="physical-space-and-other-logistics"></a>物理的なスペースも他の物流  
 物理領域は、計画段階中に考慮すべき重要な詳細です。 ラボを実行するためにコンピューターがアクセスできる、参加者に動作し、ディスカッションのスペースは必要がありますが存在する必要があります。 参加者が電話会議を小さいパブリック フォーラムで保持されることも、追加の領域も役立ちます。  
  
 ラボの物理環境の最も重要な側面の 1 つは、すべての参加要素の画面の同じセットの参照をできます快適環境を作成します。 これは、チームがラボの中に使用される、リモート デスクトップ セッションを表示するプロジェクターを使用して、またはリモート デスクトップの「シャドウ」機能を使用してセッション シャドウ処理によって実現できます。  
  
 パフォーマンス評価の契約、緊密なスケジュールで、engagement は、通常ファクトによってのみ一層複雑化が予期しない問題が生じる。 マイルス トーンに対応するために必要な時間がかかる場合がありますの作業日、により物理領域の物流担当は、engagement の成功に重要になります。  
  
 要約すると、パフォーマンス評価に必要な物理容量を計画するときに、次の要因を検討してください。  
  
-   ラボのコンピューターは、チーム メンバーにアクセス可能である必要があります。  
  
-   チーム メンバーは、スペース、ラボの中にディスカッションを保持する必要があります。  
  
-   チーム メンバーは、通常、オーバーヘッド プロジェクターまたはリモート デスクトップの「シャドウ」機能を使用してリモート デスクトップの画面に関連付けられている、画面の同じセットを表示するすべてのメンバーは、ビジュアルのメディアへのアクセスが必要です。
