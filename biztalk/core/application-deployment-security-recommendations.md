# <a name="application-deployment-security-recommendations"></a>アプリケーションの展開のセキュリティに関する推奨事項
次に、環境内での BizTalk アプリケーションを展開するためのガイドラインを示します。  
  
-   すべての随意アクセス制御リスト (Dacl) は、アプリケーションを .msi ファイルにエクスポートするときに、ファイルとフォルダーから削除されます。 .Msi ファイルからアプリケーションをインストールした後は、ファイルとフォルダーのすべてのセキュリティ設定を再構成する必要があります。  
  
-   セキュリティ上の理由により、BizTalk Server では、バインド ファイルのエクスポート時にバインドのパスワードがファイルから削除されます。 バインドをインポートした後で、送信ポートと受信場所のパスワードを再構成する必要があります。 BizTalk Server 管理コンソールの [トランスポートのプロパティ] ダイアログ ボックスで、送信ポートと受信場所のパスワードをそれぞれ構成します。 手順については、次を参照してください。[送信ポートを作成する方法](../core/how-to-create-a-send-port2.md)します。 参照してください[を作成する方法、受信場所](../core/how-to-create-a-receive-location.md)します。 バインド ファイルの詳細については、次を参照してください。[バインド ファイルとアプリケーションの展開](../core/binding-files-and-application-deployment.md)します。  
  
-   プロパティ スキーマの展開または展開解除は機密データの公開につながる可能性があり、追跡時に機密情報が公開されてしまう可能性もあります。 プロパティ スキーマが含まれているアセンブリを展開または展開解除すると、必ずイベント ビューアーによってイベントが Windows アプリケーション イベント ログに記録されます。 イベント ログに記録されているメッセージを調べて、すべてのアセンブリ展開アクティビティが機密データに関するポリシーに従っていることを確認する必要があります。  
  
     デプロイのイベント ログに生成されるメッセージです。  
  
     **ユーザー"{1}「アセンブリの展開」{0}"プロパティのスキーマを含んでいます。**  
  
     展開解除は、イベント ログに記録されるメッセージは次のとおりです。  
  
     **ユーザー"{1}「アセンブリの展開を解除」{0}"プロパティのスキーマを含んでいます。**  
  
-   アプリケーションには、仮想ディレクトリが含まれている場合は、こと、仮想ディレクトリのセキュリティ設定されていたアプリケーションのエクスポート中に、.msi ファイルが生成されたときにあります。 アプリケーションをエクスポートする前に、運用環境にアプリケーションを展開する場合、設定がセキュリティ要件を満たしているかを確認してください。  
  
     ただし、仮想ディレクトリを含むアプリケーションを展開し、展開先の環境に仮想ディレクトリが既に存在する場合は、既存の仮想ディレクトリのセキュリティ設定が有効になります。 展開する仮想ディレクトリと一致するようには変更されません。 既存の仮想ディレクトリのセキュリティ設定が要件を満たしているかを確認する必要があります。  
  
    > [!CAUTION]
    >  仮想ディレクトリは、HTTPS (Hypertext Transfer Protocol over Secure Socket Layer) プロトコルを使用している場合、そのセキュリティ設定は、エクスポート中は保持されませんし、インポートすると、仮想ディレクトリは、ルートのセキュリティ設定を継承します。 セキュリティ設定が要件を満たしているかを確認する必要があります。  
  
-   アプリケーションをインポートするときに、Web サービスに対して指定されたアプリケーション プールが存在しない場合は、既定のアプリケーション プールが使用されます。 既定のアプリケーション プールのセキュリティ設定は、要件に適したにできません。 そのためお勧めすることをアプリケーション プールを作成し、アプリケーションをインポートする前に、適切なセキュリティ設定を構成または既定のアプリケーション プールの設定が適切であることを確認します。  
  
-   悪意のある .msi ファイル作成者によって作成できるセキュリティの問題を回避するために展開する Windows インストーラー (.msi) ファイルのソースが信頼できることを確認します。 詳細については、次を参照してください。[セキュリティと Windows インストーラー](../core/security-and-windows-installer.md)します。  
  
-   アプリケーションを展開するアクセス許可があることを確認します。 詳細については、次を参照してください。[を展開すると、BizTalk アプリケーションの管理に必要なアクセス許可](../core/permissions-required-for-deploying-and-managing-a-biztalk-application.md)します。  
  
-   重要なビジネス データ接続、構成情報などを含めることは、BizTalk 管理者のみがアセンブリ、バインド ファイル、およびポリシーのファイルへのアクセスがあることを確認します。 ネットワーク共有経由でアプリケーションを展開する場合は、BizTalk 管理者のみがその内容を表示できるように、随意アクセス制御リスト (DACL) をネットワーク共有上に構成します。 グループおよびユーザー アカウントの詳細については、次を参照してください。 [Windows グループと BizTalk Server でのユーザー アカウント](../core/windows-groups-and-user-accounts-in-biztalk-server.md)します。  
  
-   デプロイ操作を実行するときに、BizTalk Server は、BizTalk 管理データベースと通信します。 それらの間にファイアウォールが存在する場合は、処理、サービス、およびデータ ドメイン間のファイアウォールの適切なポートを開く必要があります。 詳細については、次を参照してください。 [BizTalk Server の必須ポートの](../core/required-ports-for-biztalk-server.md)します。  
  
-   リモートの場所をポイントするには、アセンブリ、バインド ファイル、または機密データを含む可能性のあるその他のリソース ファイルをソース コンピューターと展開を実行するコンピューター間のネットワーク セキュリティを検討してください。 これら 2 つのコンピューター間のネットワークが完全に分離された潜在的な攻撃者からない場合は、ターゲット ファイルをリムーバブル メディアにコピーし、物理的配置を実行するコンピューターに移動します。
