# <a name="tpe-menu-options"></a>TPE のメニュー オプション
このトピックでは、追跡プロファイル エディター (TPE) のメニュー オプションについて説明します。 メイン メニューには、ファイル、ツール、およびヘルプが含まれます。  
  
 アプリケーションの左側のウィンドウで、アクティビティ ビューに表示される項目のコンテキスト メニューで、メッセージングの傍受に固有の追加機能が公開されます。  
  
## <a name="file-menu"></a>[ファイル] メニュー  
 [ファイル] メニューには、次のオプションが含まれています。  
  
-   **新しい**– 新しい追跡プロファイルを作成します。 このオプションは常にします。  
  
-   **開いている**– 既存の追跡プロファイルが表示されます。 このオプションは常にします。  
  
-   **保存**– 現在編集中の追跡プロファイルを保存します。 追跡プロファイルが編集されていない場合はこのオプションは利用できません。  
  
-   **名前を付けて保存**– 指定した名前を現在編集中の追跡プロファイルを保存します。 追跡プロファイルが編集されていない場合はこのオプションは利用できません。  
  
-   **BAM アクティビティ定義のインポート**– 適用されているアクティビティ定義をインポートします。 このメニュー項目が、同じ機能レベルのウォーターマークのリンクと、左側のフレームにユーザーから開始できますこのアクションどちらの場所 (ある種の使いやすさのヒントをされる基準値)。 BizTalk 管理データベースおよび BAM プライマリ インポート データベースへの接続がある場合に、このオプションは使用可能な常にします。  
  
-   **終了**– TPE を終了します。 アクティブな編集時に終了する Windows の標準に従って保存または名前を付けて保存を呼び出します。 このオプションは常にします。  
  
### <a name="new-profile"></a>新しいプロファイル  
 **新しいプロファイル**メニュー オプションは空白を表示する追跡プロファイルのソース ファイルが空の BTT ファイルに対してコンテンツなし (追跡プロファイル ファイル)。 新しいプロファイルの既定の名前は、TrackingProfile1 を (... が2, 3, など。)。  
  
> [!NOTE]
>  既定では、BizTalk 管理データベースは、BizTalk グループの既定の BizTalk 管理データベースに設定されます。 管理データベースを変更するには、使用、**管理データベースの設定**オプション、**ツール**します。 BizTalk 管理データベースの設定は、セッション間で保持されます。  
  
### <a name="open"></a>[ファイル]  
 **オープン**メニュー オプションにより、ユーザーを検索して動作する 1 つのプロファイルを選択します。 プロファイルは、アセンブリに依存しているを表示したり、プロファイルの検証を行うために作成されました。  
  
 元の展開済みのアセンブリが使用できなくまたはプロファイルが関連付けられているアセンブリを変更する必要がある、明示的に呼び出すオプションがある、**管理データベースの設定**このリンクを更新する操作。 プロファイルを編集し、それを適用できます。  
  
### <a name="save"></a>保存  
 **保存**オプションは、現在編集中の追跡プロファイルを保存します。 新しい、オプションでは追跡プロファイルの保存が選択されて初めて、**名前を付けて保存**ダイアログ。  
  
### <a name="save-as"></a>[名前を付けて保存]  
 **付けて**オプションでは、フォルダーと、追跡プロファイルの名前を指定できます。 既定の拡張子は .btt 追跡プロファイル ファイル。  
  
### <a name="import-bam-activity-definition"></a>BAM アクティビティ定義のインポート  
 **BAM アクティビティ定義のインポート**オプションは、BAM プライマリ インポート データベースで見つかった BAM アクティビティ定義の一覧が設定されている BAM アクティビティ定義のインポート ダイアログ ボックスを開きます。  
  
 クリックしてこの機能が呼び出されますも**ここをクリックすると、BAM アクティビティ定義のリンクをインポートする**新しいのアクティビティ ビューで追跡プロファイル。  
  
 **既存のプロファイル チェック ボックスを取得します。**  
  
 このチェック ボックスを使用すると、抽出し、既存の追跡アクティビティ定義のプロファイルのマッピングを含めることができます。  既定では、チェック ボックスが選択されていません。  
  
> [!NOTE]
>  BAM アクティビティ定義をインポートするには、追跡の設定と、既存のマッピングは、アセンブリ、オーケストレーション、スキーマ、および現在選択されている BizTalk 管理データベース内のポートに対して検証されます。 マッピングが、データベース レプリカに対して有効でない場合は、無効なマッピングの横にあるエラー アイコンが表示されます。 これが発生するインスタンスとは、アセンブリの展開が解除する前に、または BAM プライマリ インポート データベースが複数のグループおよび複数の BizTalk 管理データベースで共有されている場所に配置された追跡プロファイルです。 後者の場合、追跡プロファイルを 1 つのグループを展開し、別のグループからインポートされた場合にエラー アイコンが表示されます。  
  
### <a name="exit"></a>終了する  
 **終了**TPE を閉じます。 追跡プロファイルの変更があった場合、追跡プロファイル エディターを閉じるときに、プロファイルの保存を求められます。  
  
## <a name="tools-menu"></a>[ツール] メニュー  
 [ツール] メニューには、次のオプションが含まれています。  
  
-   **追跡プロファイルの適用**– プロファイルを BAM プライマリ インポート データベースに格納します。  
  
-   **追跡プロファイルを削除する**– 追跡を削除します。 BAM プライマリ インポート データベースからのマッピングをプロファイルします。  
  
-   **管理データベースの設定**– TPE がに対して作業している管理データベースを設定します。  
  
-   **オプション**: TPE オプションを設定することができます。  
  
### <a name="apply-tracking-profile"></a>追跡プロファイルを適用します。  
 **追跡プロファイルの適用**メニュー オプションが指定されたアセンブリにも対応するオーケストレーションとポート マッピングの追跡プロファイルを格納します。 追跡プロファイルで参照されるすべての成果物の BizTalk 管理データベースを指定できるようにします。 関連するサービスのすべての新しいインスタンスは、開始時に、新しい追跡プロファイルを使用します。  
  
 同時に複数のアセンブリに対して操作を行えます。 連携していないアセンブリで同時が、複数のアセンブリは、任意の 1 つの BAM アクティビティ定義の基準とした単一の編集セッションでアクセスできます。  つまり、BAM アクティビティ定義をインポートしてから、追跡された項目を描画する 3 つのアセンブリを選択して場合、TPE を作成するプロファイルが、オーケストレーションと追跡に使用されるポートに関連付けられている複数のプロファイルに適用されます。プロファイル。 追跡プロファイルには、常に、特定の BAM アクティビティ定義に関連付けられているすべてのアセンブリに関する情報が含まれています。  
  
> [!CAUTION]
>  BizTalk Server では、各 BAM アクティビティは 1 つの追跡プロファイルに対して展開のみを持つことができます。 つまり、オーケストレーションにマップされているアクティビティを含むプロファイルの展開し、後で、別のオーケストレーションにマップされた同じアクティビティを含むさまざまなプロファイルを展開すると、最初に展開されたプロファイル経由で書き込まれます。  
  
### <a name="remove-tracking-profile"></a>追跡プロファイルを削除します。  
 **追跡プロファイルの削除**メニュー オプションが読み込まれたアクティビティ定義とその対応するオーケストレーションとポートの追跡プロファイルを削除します。 この操作を完了する前に、確認を求められます。  
  
### <a name="set-management-database"></a>管理データベースの設定  
 **管理データベースの設定**メニュー オプションをマッピングからソースの選択し、なり、bam アクティビティ定義にある BizTalk 管理データベースを指定します。 既定では、追跡プロファイルは、構成時に指定された既定の BizTalk 管理データベースに設定されます。  
  
 SQL 接続文字列は、2 つの方法のいずれかで指定できます。  
  
-   サーバー名だけを指定して、TPE は既定のポートに接続されます。  
  
-   サーバー名を指定して、ポートの組み合わせ、つまり、ポート番号。 TPE は、指定したポートを使用している SQL Server を実行するコンピューターに接続されます。  
  
## <a name="help-menu"></a>Help(ヘルプ) メニュー  
 [ヘルプ] メニューには、次のオプションが含まれています。  
  
-   **BizTalk Server ヘルプ**– BizTalk Server ヘルプ ファイルを開きます。  
  
-   **追跡プロファイル エディター ヘルプ**– BizTalk Server のヘルプ ファイルの TPE に関するセクションを開きます。  
  
-   **BizTalk Server オンライン**-BizTalk Server のオンライン ヘルプを開きます。  
  
-   **追跡プロファイル エディターについて**– 標準のバージョン情報が表示されます。
