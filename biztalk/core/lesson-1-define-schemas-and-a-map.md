# <a name="lesson-1-define-schemas-and-a-map"></a>レッスン 1:スキーマおよびマップを定義します。
このレッスンでは、作成し、エンタープライズ アプリケーション統合 (EAI) ソリューションの最初のプロジェクトをビルドします。 プロジェクトには、2 つのメッセージ スキーマおよびマップが含まれています。  
  
 EAI ソリューションで倉庫システムに送信する在庫補充要求メッセージを[!INCLUDE[btsBizTalkServerNoVersion](../includes/btsbiztalkservernoversion-md.md)]処理します。 このレッスンでは、次の項目を作成します。  
  
- EAI ソリューション、プロジェクトを保持するためにします。  
  
- スキーマとマップを保持するプロジェクトです。  
  
- 倉庫に送信メッセージのスキーマ、[!INCLUDE[btsBizTalkServerNoVersion](../includes/btsbiztalkservernoversion-md.md)]在庫の補充を要求します。  
  
- 要求が拒否されたときに倉庫で予期されるメッセージのスキーマです。  
  
- 要求拒否メッセージを作成する要求メッセージを再フォーマットのマップ。  
  
  最後に開始する前にプロジェクトをビルドする[レッスン 2。ビジネス プロセスの定義](../core/lesson-2-define-the-business-process.md)します。 レッスン 2 では、メッセージをルーティングし、承認基準に照らし合わせて、在庫補充要求メッセージの内容を評価するビジネス プロセスを作成します。
