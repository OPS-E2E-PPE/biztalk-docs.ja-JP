# <a name="as2-message-and-correlated-mdn-status-report"></a>AS2 メッセージおよび関連する MDN 状態レポート
このレポートでは、それらのインターチェンジに関連する Mdn と、AS2 で処理される AS2 メッセージが送信および受信パイプラインをすべて表示されます。  
  
## <a name="fields-in-the-status-report"></a>状態レポート内のフィールド  
 AS2 メッセージおよび関連する ACK の状態レポートは、それらを受信または送信したインターチェンジと関連付けられる受信確認の次の情報が表示されます。  
  
-   送信者パーティ名  
  
-   受信者パーティ名  
  
-   As2 の送信元  
  
-   AS2 の宛先  
  
-   AS2 メッセージ ID  
  
-   AS2 メッセージの日時  
  
-   受信日時  
  
-   パーティ ロール  
  
-   MDN の状態  
  
-   暗号化の状態  
  
-   署名の状態  
  
-   Edi インターチェンジの状態  
  
-   AS2 メッセージを重複しています  
  
-   重複をチェックする日数  
  
-   Filename  
  
-   信頼できるメッセージングを有効になっています。

## <a name="additional-as2-message-and-correlated-mdn-status-reports"></a>その他の AS2 メッセージおよび関連する MDN 状態レポート  
 AS2 メッセージおよび関連する MDN 状態レポートに表示されている AS2 メッセージを右クリックした場合は、1 つを表示、次の状態レポートの詳細。  
  
|状態レポート|表示される状態|  
|-------------------|----------------------|  
|インターチェンジ/確認の状態|AS2 メッセージの EDI ペイロードの状態|  
|再送信状態の詳細|メッセージの再送信の状態|  
|メッセージのワイヤ形式|AS2 メッセージのワイヤ形式|  
|デコードされたメッセージ|AS2 メッセージのデコード形式|  
|MDN メッセージ|AS2 メッセージに関連する MDN メッセージ|  
  
 最後の 3 つのメッセージのそれぞれの形式は同じです。 詳細については、次を参照してください。 [AS2 メッセージ コンテンツ状態レポート](../core/as2-message-content-status-reports.md)します。  
  
## <a name="correlate-an-as2-message-with-its-edi-payload"></a>その EDI ペイロードを持つ AS2 メッセージを関連付ける  
 AS2 および EDI 状態レポートには、AS2 メッセージとその EDI ペイロードの状態エントリを関連付けることができます。 両方の AS2 および EDI 状態レポートが有効にした場合、AS2 メッセージの AS2 状態レポートで状態エントリが行われ、その AS2 メッセージの EDI ペイロードの EDI 状態レポートで、状態エントリが作成します。 AS2 メッセージの状態エントリを右クリックし、をクリックして AS2 メッセージの EDI ペイロードの状態を表示するには、AS2 状態レポートの表示があれば、**インターチェンジ/確認の状態**します。 AS2 メッセージに EDI インターチェンジの 1 つだけがある場合、このアクションはその 1 つのインターチェンジの状態エントリが表示されます。  
  
 AS2 メッセージには、複数の EDI インターチェンジが含まれている AS2 メッセージ内の複数の EDI インターチェンジの状態を表示しようとした場合は、最後の EDI インターチェンジだけがインターチェンジ/確認の状態レポートに表示されます。 さらに、 **EDI のインターチェンジ制御番号**AS2/MDN の状態レポート内のフィールドは、最後のインターチェンジ制御番号のみを表示します。 (インターチェンジ制御番号によって、AS2 メッセージとその EDI インターチェンジ ペイロードです。)  
  
 すべての EDI インターチェンジを AS2 メッセージのデータは、状態のレポート データベースに保存されます。 場合、インターチェンジ/確認の状態レポートで AS2 メッセージのすべてのインターチェンジを表示することができます、 **Control ID Equals All**句は、状態レポート クエリでは。 AS2 メッセージに含まれていないその他のインターチェンジも表示されます。れる EDI インターチェンジを決定するただし、1 つの AS2 メッセージでは、送信者パーティ、受信者パーティ、およびインターチェンジの日時などの他のフィールドでします。  
  
 **パーティ ロール**AS2 状態レポートにフィールドと**方向**EDI 状態レポートにフィールドの意味は正反対です。 EDI ペイロードを持つ受信 AS2 メッセージの**パーティ ロール**フィールドは、AS2 メッセージになります**送信者**中、**方向**フィールドは、EDI インターチェンジとなります**受信**します。 この理由は、受信メッセージのパーティから受信した場合のパーティのロールが、送信者が。 そのパーティのプロパティは、送信 AS2 メッセージの送信者としてのパーティは、パートナー アグリーメント マネージャーでの AS2 のプロパティ ダイアログ ボックスで定義されている AS2 メッセージに関連します。 結果として、AS2 パーティのロールは、EDI の方向と逆です。
