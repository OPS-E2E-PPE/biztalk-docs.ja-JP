# <a name="how-to-configure-an-application"></a>アプリケーションを構成する方法
このトピックでは、BizTalk Server 管理コンソールを使用して、アプリケーションのアイテムを次のように構成する方法について説明します。  
  
-   論理ポートおよびロール リンクを物理ポートおよびパーティにバインドします。  
  
-   オーケストレーションを実行するときに使用するホストにオーケストレーションをマップします。  
  
-   送信ポート、受信ポート、送信ポート グループ、および受信ポートの場所を構成します。  
  
 アプリケーションに 1 つ以上のオーケストレーション、送信ポート、送信ポート グループ、受信ポート、または受信場所が含まれていない場合、このトピックの情報は適用できません。  
  
 この構成を完了すると、ことができます、アプリケーションを起動する」の説明に従って[起動し、BizTalk アプリケーションを停止する方法](../core/how-to-start-and-stop-a-biztalk-application.md)です。  
  
> [!NOTE]
>  アプリケーション内でアイテムを個別に構成することもできます。 個別のアイテムの構成の詳細については、このトピックの最後のリンクを参照してください。  
  
## <a name="prerequisites"></a>前提条件  
 このトピックの手順を実行するには、BizTalk Server 管理者グループのメンバーであるアカウントを使用してログオンする必要があります。 詳細なアクセス許可についてを参照してください。[を展開すると、BizTalk アプリケーションの管理に必要なアクセス許可](../core/permissions-required-for-deploying-and-managing-a-biztalk-application.md)です。  
  
### <a name="to-configure-an-application"></a>アプリケーションを構成するには  
  
1.  をクリックして**開始**、 をポイント**プログラム**、 をポイント[!INCLUDE[btsBizTalkServerStartMenuItemui](../includes/btsbiztalkserverstartmenuitemui-md.md)]、順にクリック**BizTalk Server 管理コンソール**です。  
  
2.  コンソール ツリーで  **BizTalk Server 管理コンソール**アプリケーションを右クリックし、クリックして**構成**です。  
  
3.  **オーケストレーション**左側のウィンドウに一覧表示、オーケストレーションをクリックして、次の表の説明に従ってプロパティを構成、およびをクリックして**OK**です。 このアプリケーションにオーケストレーションが存在しない場合、**オーケストレーション**一覧は表示されません。  
  
    |プロパティ|目的|  
    |--------------|----------------|  
    |**ホスト**|ドロップダウン リストから、アプリケーションを参加させるホストを選択します。|  
    |**[バインド]-[受信論理ポート]**|論理受信ポートの名前を表示します。 論理ポートはオーケストレーションのポート画面に表示されます。|  
    |**[バインド] - 受信ポート**|ドロップダウン リストから、対応する受信論理ポートにバインドする物理受信ポートを選択します。 この一覧には、現在のアプリケーション内のポート、および任意の参照先アプリケーション内のポートが含まれます。|  
    |**[バインド]-[送信論理ポート]**|論理送信ポートの名前を表示します。 論理ポートはオーケストレーションのポート画面に表示されます。|  
    |**バインド-送信ポート/送信ポート グループ**|ドロップダウン リストから、対応する送信論理ポートにバインドする送信ポートまたは送信ポート グループを選択します。 この一覧には、現在のアプリケーション内のポート、および任意の参照先アプリケーション内のポートが含まれます。|  
  
4.  **メッセージング**左側のウィンドウの一覧で、をクリックして**送信ポートおよび送信ポート グループ**の次の表の説明に従ってプロパティを構成し、クリックして**OK**です。 この一覧には、現在のアプリケーションのすべての送信ポートおよび送信ポート グループが表示されます。  
  
    |プロパティ|目的|  
    |--------------|----------------|  
    |**新規**|新しい送信ポートまたは送信ポート グループを作成する場合にクリックします。 オプションの送信ポートは、右クリックすると、使用可能なオプションと同じ、**送信ポート**ポイントすると、コンソール ツリー ノード**新規**です。 詳細については、次を参照してください。[送信ポートを作成する方法](../core/how-to-create-a-send-port2.md)です。 参照してください[送信ポート グループを作成する方法](../core/how-to-create-a-send-port-group.md)です。|  
    |**プロパティ**|選択した送信ポートまたは送信ポート グループのプロパティを表示する場合にクリックします。 送信ポートを選択する場合、**送信ポートのプロパティ**ページが表示されたら、ポートを構成できます。 送信ポート グループを選択した場合、**送信ポート グループのプロパティ**ページが表示されたら、送信ポート グループを構成することができます。 プロパティの構成の詳細については、次を参照してください。[管理 送信ポートと送信ポート グループ](../core/managing-send-ports-and-send-port-groups.md)です。|  
    |**名前**|物理送信ポートの名前を表示します。|  
    |**Assert**|送信ポートに適用されるフィルター式を表示します。 フィルターが設定されて、**送信ポートのプロパティ**ウィンドウです。|  
    |**URI**|選択したポートの URI を表示します。|  
  
5.  **メッセージング**左側のウィンドウの一覧で、をクリックして**ポートと受信場所**の次の表の説明に従ってプロパティを構成し、クリックして**OK**です。 この一覧には、現在のアプリケーションのすべての受信ポートおよび受信場所が表示されます。  
  
    |プロパティ|目的|  
    |--------------|----------------|  
    |**新規**|新しい受信ポートを作成する場合にクリックします。 オプションを新しい受信ポートは、右クリックすると、使用可能なオプションと同じ、**受信ポート**ポイントすると、コンソール ツリー ノード**新規**です。 詳細については、次を参照してください。[受信ポートを作成する方法](../core/how-to-create-a-receive-port.md)です。|  
    |**プロパティ**|クリックすると選択したアイテムのプロパティが表示されます。 受信ポートを選択している場合、**受信ポートのプロパティ**ページが表示されたら、ポートを構成できます。 受信場所を選択した場合、**受信場所のプロパティ**ページが表示されたら、受信場所を構成することができます。 選択するには、受信ポートの右側にある太字のテキストをクリックして**受信ポート**です。 プロパティの構成の詳細については、次を参照してください。[管理 受信ポート](../core/managing-receive-ports.md)です。 参照してください[受信場所の管理](../core/managing-receive-locations.md)です。|  
    |**名前**|受信ポートに関連付けられている受信場所の名前を表示します。|  
    |**URI**|選択した受信場所の URI を表示します。|  
  
6.  左側のウィンドウでをクリックして**ロール リンク**次の表で説明するプロパティを構成し、クリックして**OK**です。 アプリケーションでは、ロール リンクがない場合、**ロール リンク**リンクは表示されません。  
  
    |プロパティ|目的|  
    |--------------|----------------|  
    |**参加させる**|クリックすると表示、**パーティを参加させる**ダイアログ ボックスで、ロールによって定義されるインターチェンジに参加するパーティを選択できます。|  
    |**参加解除します。**|選択したパーティをインターチェンジから参加解除する場合にクリックします。|  
    |**バインド**|ロール リンクと物理ポート間の接続をパーティを介して確立する場合にクリックします。|  
    |**パーティ**|このロールに参加させたパーティの名前を表示します。|
