# <a name="batch-status-report"></a>バッチの状態レポート
このレポートでは、バッチ オーケストレーションのインスタンスのアクティビティを示します。 レポートの各行には、バッチ オーケストレーションの単一のインスタンスによって処理されているバッチの状態が示されます。  
  
 保存されたバッチの状態は、バッチの状態レポートでは報告されませんが、インターチェンジ/確認の状態レポートでは報告されます。  
  
## <a name="fields-in-the-status-report"></a>状態レポート内のフィールド  
 バッチの状態レポートには、バッチ化されたインターチェンジについての次の情報が表示されます。  
  
-   バッチの状態  
  
-   バッチ名  
  
-   送信先パーティ名  
  
-   アクティベーション時間  
  
-   バッチの発生  
  
-   EDI エンコードの種類  
  
-   バッチの種類  
  
-   バッチの対象  
  
-   バッチ要素数: バッチ要素の数によって累積されたバッチ処理オーケストレーション インスタンス  
  
-   拒否された要素数  
  
-   バッチがアクティブ化、スケジュールされたリリース、カウントに基づくリリース、手動の上書きリリース、バッチの終了停止/リリース、要素を追加、または外部リリースできる最新のアクション。  
  
-   インターチェンジ制御番号  
  
-   インターチェンジの日時  
  
-   バッチ サイズ (文字) (既定では表示されません)  
  
## <a name="view-related-interchanges-status-reports"></a>View Related Interchanges の状態レポート  
 バッチの状態レポートに一覧表示されるインターチェンジまたは確認を右クリックすると、そのバッチに関連するインターチェンジの詳細が表示されます。
