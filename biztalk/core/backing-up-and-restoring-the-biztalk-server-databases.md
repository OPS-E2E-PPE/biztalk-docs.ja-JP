# <a name="backing-up-and-restoring-the-biztalk-server-databases"></a>バックアップおよび BizTalk Server データベースを復元します。
このセクションでは、バックアップして、BizTalk Server データベースを復元する方法について説明します。 ハードウェア障害が発生した場合、一貫性のある BizTalk Server 環境を復元することを確認するには、このセクションの手順が従う必要があります。 BizTalk Server では、重要なは、バックアップを作成し、すべてのデータベースを復元するために、データベース間で分散トランザクションを実行します。  
  
 BizTalk Server では、カスタマイズされたバックアップ プロセスとトランザクション ログ マーキングと共にデータベースの完全バックアップとログ バックアップを使用する必要があります。 このプロセスの詳細については、次を参照してください。[マークされたトランザクション、完全バックアップ、およびログ バックアップ](../core/marked-transactions-full-backups-and-log-backups.md)します。
