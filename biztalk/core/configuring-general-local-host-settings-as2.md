### <a name="to-configure-the-general-settings"></a>全般的な設定を構成するには  
  
1.  AS2 アグリーメントを作成する」の説明に従って[全般設定の構成 (AS2)](../core/configuring-general-settings-as2.md)です。 既存の AS2 アグリーメントを更新するでアグリーメントを右クリックし、**パーティとビジネス プロファイル** ページで、をクリックして**プロパティ**です。  
  
2.  一方向アグリーメント タブで、**ローカル ホストの設定**セクションで、**全般**です。  
  
3.  選択、**メッセージを送信する前に証明書失効リストを確認して**送信メッセージの署名に使用される証明書を証明書失効リストに含まれているかどうかを決定する チェック ボックスを示すit が失効していない、または有効期限の日付が渡されるかどうか。 証明書が証明書失効リストに含まれている場合または期限切れの場合、BizTalk Server はメッセージを暗号化せずに保留します。 このプロパティをオフにした場合、この確認は行われません。  
  
    > [!NOTE]
    >  証明書失効リストを取得および検索する場合には遅延が生じます。  
  
4.  **既定のコンテンツの種類**、送信 AS2 メッセージのパーティに使用する既定のコンテンツ タイプを選択します。 場合、`ContentType`プロパティをこの値の設定は、送信メッセージの生成に使用されます。 それ以外の場合、メッセージのボディ部のコンテキストで設定**既定のコンテンツの種類**プロパティを使用します。  
  
5.  選択、 **MIME ヘッダーで送信ファイル名**送信 AS2 メッセージの MIME ヘッダーの一部として、ファイル名を送信する チェック ボックスです。 ファイル名を指定するには選択**ファイル名の指定**または**システム ファイル名を生成している**です。  
  
    > [!NOTE]
    >  選択すると**システム ファイル名を生成している**AS2 メッセージで送信される各添付ファイルのファイル名と、新しい GUID 値が生成されます。  
  
6.  選択した場合は**ファイル名の指定**で文字列値またはコンテキスト プロパティを入力して、**ファイル名の指定**フィールドです。 有効にすることも**コンテキスト プロパティ (%property% など) が見つからない場合は、メッセージを保留**送信メッセージの選択したコンテキスト プロパティが存在しない場合は、メッセージを中断します。  
  
    > [!NOTE]
    >  コンテキスト プロパティを指定するには、プロパティ名を % 文字で囲みます。 たとえば、 `%FILE.ReceivedFileName%`のようにします。  
  
7.  をクリックして**適用**構成を続行する前に、変更を受け入れるか、をクリックする**OK**を変更を検証し、ダイアログ ボックスを閉じます。
