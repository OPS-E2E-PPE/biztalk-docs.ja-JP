# <a name="source-control-and-the-biztalk-project-system"></a>ソース管理と、BizTalk プロジェクト システム
ソース管理とは、ファイルのバージョンの格納や追跡を行ったり、ファイルへのアクセスを制御するメカニズムです。このメカニズムは、ソフトウェアの主要部分によって実行されます。 ソース管理では、次の操作が行えます。  
  
-   重要なファイルが上書きされる可能性を軽減する。  
  
-   ファイルにバージョン番号を適用する。  
  
-   ソース管理の対象であるファイルの古いバージョンをアーカイブする。  
  
-   ファイルの編集者、編集日時および編集内容を記録する。  
  
 Visual Studio は、MSSCCI (Microsoft Source Code Control Interface) に準拠するシステムと統合できます。 BizTalk Server は、Team Foundation Server のサポートを提供でき、開発チームが、統合ソース管理、バグの追跡、チーム開発サポートを自動化するためのサポートと Microsoft Office Project Server 統合を活用するにはMSBuild を使用してビルドします。 BizTalk プロジェクト システムは Microsoft Visual SourceSafe もサポートしています。  
  
 ソース管理の詳細については、「ソース コントロールの基本事項」で Visual Studio 連結ヘルプ コレクションを参照してください。 [http://go.microsoft.com/fwlink/?LinkId=25506](http://go.microsoft.com/fwlink/?LinkId=25506)です。
