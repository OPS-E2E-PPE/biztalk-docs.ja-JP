 <span data-ttu-id="abc7c-114">システムが実稼働の準備ができていることを確認するには、持続可能な方法のエンド ツー エンドでテストされていることを確認する必要があります。</span><span class="sxs-lookup"><span data-stu-id="abc7c-114">To ensure that the system is ready for production, you must verify that it has been tested in a sustainable fashion end to end.</span></span> <span data-ttu-id="abc7c-115">つまり、すべての操作アクティビティなどのデータベース メンテナンス、クエリを実行して、計画的な操作と計画外の停止をテストする必要があります、トピックで定義されている[維持可能なパフォーマンスとは何ですか?](../core/what-is-sustainable-performance.md)</span><span class="sxs-lookup"><span data-stu-id="abc7c-115">This means that all operations activities such as database maintenance, operations querying, and planned and unplanned outages must be tested, as defined in the topic [What Is Sustainable Performance?](../core/what-is-sustainable-performance.md)</span></span> <span data-ttu-id="abc7c-116">これは、最終テスト段階で維持可能なパフォーマンス テストのスイート全体を結合することが重要であるため、システムの準備ができてを証明する最後のチャンスです。</span><span class="sxs-lookup"><span data-stu-id="abc7c-116">This is the last chance to certify readiness for the system, so it is important to combine the full suite of sustainable performance tests in the final test pass.</span></span>
