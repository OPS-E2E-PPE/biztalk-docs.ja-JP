## <a name="controlling-send-port-priority"></a><span data-ttu-id="68bdf-117">送信ポートの優先度の制御</span><span class="sxs-lookup"><span data-stu-id="68bdf-117">Controlling Send Port Priority</span></span>  
 <span data-ttu-id="68bdf-118">トランスポートの詳細オプションの優先度設定で、メッセージ ボックスから削除するメッセージの順序を制御できます。</span><span class="sxs-lookup"><span data-stu-id="68bdf-118">The Priority setting of the Transport Advanced Options controls the order in which messages are removed from the message box.</span></span> <span data-ttu-id="68bdf-119">優先度を高く設定したポートは優先度の低いポートよりも先に処理されます。つまり、1 つのホスト内で、優先度の高いポートは他の送信ポートと比べ重要度が高くなります。</span><span class="sxs-lookup"><span data-stu-id="68bdf-119">Ports with a higher priority will be processed earlier than ports with a lower priority making the higher priority ports more important relative to other send ports within a single host.</span></span>  
  
 <span data-ttu-id="68bdf-120">特定の種類の要求に短時間で応答する必要がある場合などは、優先度が役立ちます。</span><span class="sxs-lookup"><span data-stu-id="68bdf-120">Priority is useful in scenarios requiring low response times for certain types of requests.</span></span> <span data-ttu-id="68bdf-121">たとえば、複数の送信ポートを使用し、通常の要求を処理する場合と対話型の要求を処理する場合で異なるシステムに接続するとします。</span><span class="sxs-lookup"><span data-stu-id="68bdf-121">For example, if there are multiple send ports that connect to different systems for processing normal requests and interactive requests.</span></span> <span data-ttu-id="68bdf-122">対話型の要求には短時間で応答する必要があるため、対話型の要求が送信された場合はできるだけ早く処理する必要があります。</span><span class="sxs-lookup"><span data-stu-id="68bdf-122">The interactive requests require a low response time, so when an interactive request is submitted, you want to ensure it is processed as soon as possible.</span></span>  
  
 <span data-ttu-id="68bdf-123">メッセージ ボックスに優先度の異なるメッセージがある場合、BizTalk Server で均一な処理は行われません。</span><span class="sxs-lookup"><span data-stu-id="68bdf-123">BizTalk Server does not attempt to be fair in the processing of messages with different priorities in the message box.</span></span> <span data-ttu-id="68bdf-124">処理の開始時、メッセージ ボックス内の項目に設定されている優先度が 2 種類あり、それぞれの項目数が同じ場合は、優先度の高い項目がすべて処理された後、優先度の低い項目が処理されます。</span><span class="sxs-lookup"><span data-stu-id="68bdf-124">If there are equal numbers of items with two different priorities in the message box when processing begins, the lower priority items will be processed only after all high priority items have been processed.</span></span> <span data-ttu-id="68bdf-125">優先度の高い項目の数が非常に多い場合は、優先度の低い項目は処理されない可能性があります。</span><span class="sxs-lookup"><span data-stu-id="68bdf-125">If the volume of high priority items is great, it is possible that items with a lower priority will never be processed.</span></span> <span data-ttu-id="68bdf-126">つまり、優先度の低い項目に対してリソースが不足します。</span><span class="sxs-lookup"><span data-stu-id="68bdf-126">In other words, the lower priority items will experience starvation.</span></span>  
  
> [!WARNING]
>  <span data-ttu-id="68bdf-127">メッセージに対するリソース不足の危険性を最小限に抑えるには、実際の負荷状況下でアプリケーションを十分にテストし、すべてのメッセージが処理されるかどうかを確認します。</span><span class="sxs-lookup"><span data-stu-id="68bdf-127">To minimize the risk of message starvation, thoroughly test your application under realistic loads to ensure all messages are processed.</span></span> <span data-ttu-id="68bdf-128">ソリューションをテストしておかないと、処理されないメッセージが発生する可能性があります。</span><span class="sxs-lookup"><span data-stu-id="68bdf-128">Failure to test your solution may result in unprocessed messages.</span></span>  
  
 <span data-ttu-id="68bdf-129">BizTalk Server は内部的に、すべてのサブスクリプションに優先度を割り当てます。</span><span class="sxs-lookup"><span data-stu-id="68bdf-129">Internally, BizTalk Server assigns a priority to every subscription.</span></span> <span data-ttu-id="68bdf-130">優先度は 1 (最も高い優先度) から 10 (最も低い優先度) までの任意の数字になります。</span><span class="sxs-lookup"><span data-stu-id="68bdf-130">Priority can be any number from 1 (highest priority) to 10 (lowest priority).</span></span> <span data-ttu-id="68bdf-131">既定の優先度としては、サブスクリプションのアクティブ化には優先度 7、サブスクリプションの関連付けには優先度 5 が割り当てられます。したがって、関連付けメッセージはサブスクリプションのアクティブ化メッセージよりも先に配信されます。</span><span class="sxs-lookup"><span data-stu-id="68bdf-131">Because the default priority is 7 for activating subscriptions and 5 for correlating subscriptions, correlating messages will be delivered earlier than activating subscriptions.</span></span>
