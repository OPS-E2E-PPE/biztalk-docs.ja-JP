## <a name="introduction-to-as2"></a><span data-ttu-id="c93a4-104">AS2 の概要</span><span class="sxs-lookup"><span data-stu-id="c93a4-104">Introduction to AS2</span></span>  
 <span data-ttu-id="c93a4-105">EDI のトランスポートには、一般的に付加価値通信網 (VAN) が使用されます。</span><span class="sxs-lookup"><span data-stu-id="c93a4-105">A common transport used for EDI is Value-Added Networks (VANs).</span></span> <span data-ttu-id="c93a4-106">これらは、正確かつ法的拘束力のある監査記録などの付加価値サービスを提供するプライベート ネットワークです。</span><span class="sxs-lookup"><span data-stu-id="c93a4-106">These are private networks that provide value-added services, such as precise and legally binding audit trails.</span></span> <span data-ttu-id="c93a4-107">しかし、企業は EDI ドキュメントの交換をインターネット上で行うように移行しています。</span><span class="sxs-lookup"><span data-stu-id="c93a4-107">However, companies are migrating to exchanging EDI documents over the Internet.</span></span> <span data-ttu-id="c93a4-108">この方法は、コスト削減、柔軟性や効率性の向上を実現し、冗長性やスケーラビリティの面でも利点があります。</span><span class="sxs-lookup"><span data-stu-id="c93a4-108">This lowers costs, increases flexibility and efficiency, and has advantages in terms of redundancy and scalability.</span></span>  
  
 <span data-ttu-id="c93a4-109">EDI over the Internet (EDIINT) を実装するための最も一般的な方法として、AS2 (Applicability Statement 2) を使用する方法が挙げられます。</span><span class="sxs-lookup"><span data-stu-id="c93a4-109">The most common way of implementing EDI over the Internet (EDIINT) is by AS2 (Applicability Statement 2).</span></span> <span data-ttu-id="c93a4-110">AS2 の仕様では「MIME-Based Secure Peer-to-Peer Business Data Interchange」が定義されています。</span><span class="sxs-lookup"><span data-stu-id="c93a4-110">The AS2 specification defines MIME-based secure peer-to-peer business data interchange.</span></span> <span data-ttu-id="c93a4-111">MIME データで作成されたエンベロープを含むメッセージは、HTTP over TCP/IP を使用して送信されます。</span><span class="sxs-lookup"><span data-stu-id="c93a4-111">Messages containing an envelope with MIME data are transmitted using HTTP over TCP/IP.</span></span>  
  
 <span data-ttu-id="c93a4-112">AS2 は HTTP POST 操作を使用して EDI や XML などのビジネス データを送信します。</span><span class="sxs-lookup"><span data-stu-id="c93a4-112">AS2 uses the HTTP POST operation to send EDI, XML, or other business data.</span></span> <span data-ttu-id="c93a4-113">AS2 が実行する処理は EDI データの送信だけではありません。</span><span class="sxs-lookup"><span data-stu-id="c93a4-113">AS2 is not restricted to sending EDI data.</span></span> <span data-ttu-id="c93a4-114">要求 URI は、メッセージ データのアンパックやハンドリングに使用する AS2 処理を指定します。</span><span class="sxs-lookup"><span data-stu-id="c93a4-114">Request-URI identifies a process to be used to unpack and handle message data.</span></span> <span data-ttu-id="c93a4-115">Message Disposition Notification (MDN) は、元の送信者の URL に対する HTTP 応答メッセージの本文か、または新しい HTTP POST 操作によって、確認として返されます。</span><span class="sxs-lookup"><span data-stu-id="c93a4-115">A message disposition notification (MDN) is returned as an acknowledgment either in the HTTP response message body or by a new HTTP POST operation to a URL for the original sender.</span></span>  
  
 <span data-ttu-id="c93a4-116">EDI メッセージングの詳細については、次を参照してください。 [AS2 メッセージング](../core/as2-messaging.md)です。</span><span class="sxs-lookup"><span data-stu-id="c93a4-116">For more information about EDI messaging, see [AS2 Messaging](../core/as2-messaging.md).</span></span>
