# <a name="managing-agreements"></a>契約の管理
取引先アグリーメント (TPA) は、特定の B2B プロトコルでメッセージをやり取りする際に 2 つの取引パートナー間で交わされる決定的で拘束力があるアグリーメントと定義されます。 参照してください[取引先アグリーメント](../core/trading-partner-agreement.md)です。 

このトピックでは、作成、表示、編集、有効にするに、無効化、または契約を削除する方法を示します。  
  
## <a name="prerequisites"></a>前提条件  
 [!INCLUDE[btsBizTalkServerNoVersion](../includes/btsbiztalkservernoversion-md.md)] 管理者グループまたは [!INCLUDE[btsBizTalkServerNoVersion](../includes/btsbiztalkservernoversion-md.md)] B2B Operators グループのメンバーとしてログオンしている必要があります。

## <a name="view-or-change-an-agreement"></a>表示またはアグリーメントを変更します。  
  
1.  **BizTalk Server 管理コンソール****パーティ**です。 

2. **パーティとビジネス プロファイル** ページで、表示または編集する契約を結んでいる 2 つのビジネス プロファイルのいずれかを選択します。  
  
3.  **契約**セクションには、アグリーメントを右クリックし、**プロパティ**です。  
  
4.  **アグリーメントのプロパティ**、表示またはアグリーメントを編集します。 さまざまなタブおよび内のページで入力できる値について、**アグリーメントのプロパティ** ダイアログ ボックスで、次を参照してください。

    |--------------|--------------|  
    |X12 アグリーメント|[X12 固有のアグリーメント プロパティの構成](../core/configuring-x12-specific-agreement-properties.md)|  
    |EDIFACT アグリーメント|[EDIFACT 固有のアグリーメント プロパティの構成](../core/configuring-edifact-specific-agreement-properties.md)|  
    |AS2 アグリーメント|[AS2 アグリーメント プロパティの構成](../core/configuring-as2-agreement-properties.md)|  
  
5.  選択**適用**、プロパティを受け入れるかを選択する**OK**構成設定を完了します。 どちらの操作では、設定を検証します。 

## <a name="enable-or-disable-an-agreement"></a>アグリーメントを無効または有効にします。  
  
1.  **BizTalk Server 管理コンソール****パーティ**です。 

2. **パーティとビジネス プロファイル** ページで、編集する契約を結んでいる 2 つのビジネス プロファイルのいずれかを選択します。  
  
3.  **契約**セクションには、アグリーメントを右クリックし、**を有効にする**または**を無効にする**です。 
  
    > [!NOTE]
    >  アグリーメントの初回作成時は、既定で常に有効になります。 **を有効にする**オプションは、アグリーメントが無効な場合にのみ使用できます。 **を無効にする**オプションは、アグリーメントが有効な場合にのみ使用できます。  

## <a name="delete-an-agreement"></a>アグリーメントを削除します。  
1.  **BizTalk Server 管理コンソール****パーティ**です。 
  
2.  **パーティとビジネス プロファイル** ページで、削除する契約を結んでいる 2 つのビジネス プロファイルのいずれかを選択します。  
  
3.  **契約**セクションで、削除、およびを選択するアグリーメントを右クリックして**削除**です。  
  
4.  **アグリーメント削除の確認**ダイアログ ボックスで、**はい**した契約を削除します。
