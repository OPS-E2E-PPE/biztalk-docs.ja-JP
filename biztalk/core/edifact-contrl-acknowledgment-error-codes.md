# <a name="edifact-contrl-acknowledgment-error-codes"></a>EDIFACT CONTRL 受信確認エラー コード
このトピックでは、EDIFACT CONTRL 受信確認のセグメント内で使用されるエラー コードを使用します。 これらのセグメントの詳細については、次を参照してください。 [EDIFACT CONTRL 確認](../core/edifact-contrl-acknowledgment.md)します。  
  
 これらのエラーは、インターチェンジ、グループ、メッセージ、およびデータのレベルに適用されます。 サポートされているエラー、全体のインターチェンジ、グループ、またはトランザクションの発生時に、セットが拒否されます。 EDIFACT エンコード インターチェンジの「エラーを伴って承諾済み」の条件はありません。  
  
 **標準の EDIFACT のエラー コード**  
  
 次の表に、EDIFACT CONTRL ACK の UCI5 フィールドで使用されるエラー コード このテーブルは、EDIFACT 仕様で指定されたエラー コードが BizTalk Server の EDI でサポートされていることを示します、AS2、およびこれはサポートされていません。

|13|Missing|通知を必須 (またはそれ以外の場合に必要な) サービスかユーザー セグメント、データ要素、複合データ要素、またはコンポーネント データ要素がありません。|はい|  
|14|値がこの位置でサポートされていません|受信者が使用されている位置で、識別されたスタンドアロンのデータ要素、複合データ要素、またはコンポーネント データ要素の特定の値の使用がサポートしないことを通知します。 値は、関連する仕様に従った有効な場合があります、別の位置で使用されている場合にサポート可能性があります。|いいえ|

|16|構成要素が多すぎます|データ要素が多すぎますが、識別されたセグメントに含まれていること、またはコンポーネント データ要素が多すぎますが、識別された複合データ要素に含まれていることを通知します。|はい|  
|17|契約|アグリーメントが存在しない識別されたスタンドアロンのデータ要素、複合データ要素、またはコンポーネント データ要素の値を含むインターチェンジ、グループ、メッセージ、またはパッケージの受信を許可します。|いいえ|

|21|無効な文字|UNB セグメントで、インターチェンジで使用される 1 つまたは複数の文字がないこと、有効な文字構文識別子で定義されている通知が示されます。 無効な文字は、インターチェンジの識別された部分の直後に、参照されているレベル、またはその後の一部です。|はい|  
|22|サービスの無効な文字|通知をインターチェンジで使用されるサービス文字が UNA セグメントの指示に従って、有効なサービス文字ではないかはか、既定のサービス文字です。 UCS または UCD セグメントでコードを使用する場合、インターチェンジの識別された部分の直後後に無効な文字の後にします。|いいえ|  
|23|不明なインターチェンジの送信者|インターチェンジの送信者 (S002) が不明である通知します。|いいえ|  
|24|古すぎます|受信したインターチェンジまたはグループが、IA で指定された、または受信者によって制限よりも古いことを通知します。|いいえ|  
|25|テスト インジケーターがサポートされていません|識別されたインターチェンジ、グループ、メッセージ、またはパッケージのテストの処理を実行できないことを通知します。|いいえ|  
|26|重複が検出されました|以前に受信したインターチェンジ、グループ、メッセージ、またはパッケージの重複が検出されたことを通知します。 以前の転送が拒否されました。|はい|

|30|グループとメッセージ/パッケージの混在|メッセージ/インターチェンジのグループの外部でのパッケージを使用したグループが混在したことを通知します。|いいえ|  
|31|グループ内の 1 つ以上のメッセージ型|機能グループに、さまざまなメッセージ型が含まれることを通知します。|はい|  
|32|下位レベルが空|インターチェンジに、メッセージ、パッケージ、またはグループが含まれていないことを通知または、メッセージまたはパッケージをグループが含まれない。|いいえ|  
|33|メッセージ、パッケージ、またはグループの外側に無効な項目|メッセージ間、パッケージ、またはグループ間で、インターチェンジ内の無効なセグメントまたはデータ要素の通知です。 拒否は、上位レベルで報告されます。|はい|  
|34|入れ子にインジケーターが許可されていません|明示的な入れ子が使用されたことをメッセージの場所を使用することを通知します。|いいえ|  
|35|多くのデータ要素またはセグメントの繰り返し|何回もスタンドアロンのデータ要素、複合データ要素、またはセグメントを繰り返すことを通知します。|はい|  
|36|セグメント グループの繰り返しが多すぎます|通知のセグメント グループには、回数が多すぎますが繰り返されます。|はい|  
|37|文字の型が無効です。|アルファベット (コンポーネント) データ要素に 1 つまたは複数の数値文字が使用されるか、1 つ以上のアルファベット文字が数値で使用されることの通知 (コンポーネント) データ要素です。|はい|  
|38|桁区切り記号の前に数値がありません。|通知を 1 つ以上の数字の桁区切り記号の前がありません。|はい|  
|39|データ要素が長すぎます。|通知を受信したデータ要素の長さがデータ要素の説明で指定された最大長を超えています。|はい|  
|40|データ要素が短すぎます。|受信データ要素の長さがデータ要素の説明で指定された最小の長さよりも短いことを通知します。|はい|  
|41|永続的な通信ネットワーク エラー|通知は、インターチェンジの転送に使用される通信ネットワークによってな永続的なエラーが報告されました。 ネットワーク レベルで同じパラメーターを持つ同一のインターチェンジを再送信は成功しません。|いいえ|  
|42|一時的な通信ネットワーク エラー|通知は、インターチェンジの転送に使用される通信ネットワークによってな一時的なエラーが報告されました。 同一のインターチェンジの再送信が成功することがあります。|いいえ|  
|43|不明なインターチェンジの受信者|通知は、インターチェンジの受信者をネットワーク プロバイダーが不明です。|いいえ|  
|45|末尾の区切り記号|次のいずれかの通知:<br /><br /> データ要素区切り記号、コンポーネント データ要素区切り記号または繰り返しのデータ要素区切り記号、セグメント終端記号が、前に-最後の文字または<br /><br /> データ要素区切り記号の前に最後の文字は、コンポーネント データ要素区切り記号または繰り返しのデータ要素区切り記号がします。|はい|  
|46|サポートされている設定されていない文字|構文識別子で使用される文字の 1 つまたは複数設定されている文字ではなく通知が定義されています。または、コード拡張技術のエスケープ シーケンスで識別される文字セットは、受信者によってサポートされていません。|はい|  
|47|エンベロープ機能がサポートされていません|受信側では、エンベロープ構造が発生したことを示す通知がサポートされていません。|はい|
