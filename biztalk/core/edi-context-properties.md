## <a name="extracting-individual-fields-from-the-segment-context-properties"></a>セグメントのコンテキスト プロパティから個々 のフィールドの抽出  
 一部のプロパティが記述されたまたは EDI によってメッセージ コンテキストに昇格いないセグメントの文字列の一部としてのみが、個別のプロパティとして、受信パイプライン。 これは、パフォーマンス上の理由のためプロパティの昇格がパフォーマンスに影響を与えます。 たとえば、ISA セグメントの ISA5、ISA6、ISA7、ISA8、および ISA15 フィールドは、個別のプロパティとして、受信パイプラインによって昇格されますが、残りの ISA フィールドが、メッセージ コンテキストに ISA_Segment プロパティの一部としてのみ書き込まれます。 これらのプロパティの書き込みまたは昇格場合にのみ**ReuseEnvelope**受信したバッチ インターチェンジが保存されていないことを示す True に設定されていません。  
  
 (ISA、GS、UNB、UNG、または UNA)、メッセージ コンテキストに書き込まれるセグメントの 1 つの個別のフィールドを作成する必要がある場合は、既定では、このフィールドはメッセージ コンテキストに書き込まれませんが、メッセージ コンテキストに書き込むにはカスタム コンポーネントを記述する必要があります。 このカスタム コンポーネントは、セグメント フィールドを解析し、個々 のフィールドをメッセージ コンテキストに書き込む必要があります。  
  
 メッセージ強化サンプルでは、セグメントから個別のフィールドを抽出し、コンテキストに書き込むパーサーを使用する方法を示します。 このサンプルに含まれている、\<ドライブ\>: \Program Files\Microsoft BizTalk Server\SDK\Samples\EDI\MessageEnrichment します。 詳細については、次を参照してください。[メッセージ強化サンプル (BizTalk Server サンプル)](../core/message-enrichment-sample-biztalk-server-sample.md)します。
