# <a name="error-handling"></a>エラー処理
BizTalk Server メッセージング サブシステムをメッセージに続く経路には、処理および転送のいくつかのポイントが含まれます。 この経路に沿っての各ポイントで、BizTalk Server インフラストラクチャとカスタム パイプライン コンポーネントやオーケストレーションなどのアプリケーションが提供した要素の両方で障害が発生することができます。  
  
 このセクションに含まれる、よく知られている段階の処理と BizTalk Server の構成とオーケストレーションなどのアプリケーションが提供した要素の両方を介してこれらの障害モードがアドレス指定方法の一般的な障害モードをについて説明します。 障害の影響には、メッセージ、診断キャプチャして記録、および運用に関する考慮事項のディス ポジションが含まれます。  
  
 メッセージが失敗したときに発生するイベントは、エラーの場所と失敗したメッセージのルーティングおよび回復可能なインターチェンジ処理の状態に依存します。 次の表では、パイプラインとサブスクリプション エラーのエラー動作と再開場所をまとめたものです。  
  
|エラーの種類|失敗したメッセージのルーティング|回復可能なインターチェンジ処理|エラー動作|再開場所|  
|------------------|----------------------------|----------------------------------------|----------------------|---------------------|  
|パイプライン|Disabled|Disabled|パイプラインの前にメッセージが中断されます。|パイプラインの前。|  
|パイプライン|Disabled|有効|パイプラインの後にメッセージが中断されます。|パイプラインの前。 回復可能なインターチェンジ処理の詳細については、次を参照してください。[回復可能なインターチェンジ処理](../core/recoverable-interchange-processing.md)します。|  
|パイプライン|有効|Disabled|エラー レポート関連のメッセージ コンテキスト プロパティの昇格とメッセージを公開します。|メッセージは中断されません。 詳細については、失敗したメッセージのルーティングは、次を参照してください。[を使用できませんでしたメッセージ ルーティング](../core/using-failed-message-routing.md)します。|  
|パイプライン|有効|有効|エラー レポート関連のメッセージ コンテキスト プロパティの昇格とメッセージが公開されます。|メッセージは中断されません。|  
|サブスクリプション|Disabled|Disabled|メッセージはパイプラインの前に中断されます。 ルーティング エラー報告を作成します。|メッセージは、パイプラインの前に再開されます。 ルーティング エラー報告は再開できません。|  
|サブスクリプション|Disabled|有効|メッセージはパイプラインの後に中断されます。 ルーティング エラー報告を作成します。|メッセージは、パイプラインの前に再開されます。 ルーティング エラー報告は再開できません。 回復可能なインターチェンジ処理の詳細については、次を参照してください。[回復可能なインターチェンジ処理](../core/recoverable-interchange-processing.md)します。|  
|サブスクリプション|有効|Disabled|エラー レポート関連のメッセージ コンテキスト プロパティの昇格とメッセージが公開されます。 ルーティング エラー報告を作成します。|メッセージは中断されません。 ルーティング エラー報告は再開できません。 詳細については、失敗したメッセージのルーティングは、次を参照してください。[を使用できませんでしたメッセージ ルーティング](../core/using-failed-message-routing.md)します。|  
|サブスクリプション|有効|有効|エラー レポート関連のメッセージ コンテキスト プロパティの昇格とメッセージが公開されます。 ルーティング エラー報告を作成します。|メッセージは中断されません。 ルーティング エラー報告は再開できません。|  
  
 これらの多くは、モードは回復可能なインターチェンジ処理とエラー報告など、障害モードに対処するように設計された BizTalk Server の特定の機能に対応するエラーを列挙します。 その他の障害モードでは、BizTalk Server がアプリケーションの要素にエラーに関する情報を通信する方法に関するものし、さらに、カスタム パイプライン コンポーネント、アダプター、およびオーケストレーションなどのアプリケーション要素の通信方法を BizTalk Server のエラーに関する情報.
