# <a name="what-happens-to-artifacts-during-installation-and-uninstallation"></a>インストールおよびアンインストール時のアイテムの処理
このトピックでは、BizTalk Server がインストールして、アプリケーションをアンインストールする場合は、BizTalk アプリケーションに含まれるファイル ベースのアイテムを処理する方法について説明します。  
  
> [!NOTE]
>  送信ポート、送信ポート グループ、受信場所、および受信ポートは、ファイル ベースのアイテムでないし、インストールおよびアンインストールが影響を受けません。 オーケストレーション、スキーマ、マップ、およびパイプラインすべてデプロイし、管理を含む BizTalk アセンブリの一部として。  
  
-   **BizTalk アセンブリ。** インストール中に、BizTalk アセンブリ ファイルは、アセンブリがアプリケーションに追加されたときに指定されている場合、移行先パスにコピーされます。 インストールされます、アセンブリをグローバル アセンブリ キャッシュ (GAC) アセンブリが追加されたときに、このオプションが指定されました。 アプリケーションがアンインストールされると、アセンブリ ファイルは、ファイル システムから削除が、存在するが、GAC からアンインストールされていない場合、開発者にはこの操作を実行するアプリケーションには処理後のスクリプトが含まれていない場合。 必要に応じて、手動で GAC から BizTalk アセンブリをアンインストールできます。 詳細については、次を参照してください。 [GAC からアセンブリをアンインストールする方法](http://msdn.microsoft.com/library/464706a8-f902-4d05-a724-19169facd2b4)します。  
  
-   **.NET アセンブリです。** インストール中に、.NET アセンブリ ファイルは、アセンブリがアプリケーションに追加されたときに指定されている場合、移行先パスにコピーされます。 インストールされます、アセンブリ、GAC および Windows レジストリにアセンブリが追加されたときに、これらのオプションが指定されなかった場合。 アプリケーションのアンインストール時に存在し、Windows レジストリにない場合に、アセンブリ ファイルがファイル システムから削除されます。 開発者には、この操作を実行するアプリケーションに処理後のスクリプトが含まれていなければ、アセンブリは GAC または Windows レジストリから削除されません。 必要に応じて、GAC または Windows レジストリからの BizTalk アセンブリを手動で削除できます。 詳細については、次を参照してください。 [GAC からアセンブリをアンインストールする方法](http://msdn.microsoft.com/library/464706a8-f902-4d05-a724-19169facd2b4)と[BizTalk アプリケーションの設定およびその他のファイルを削除する方法](../core/how-to-remove-other-files-and-settings-for-a-biztalk-application.md)します。  
  
-   **ファイルです。** インストール中に、ファイルは、ファイル、アプリケーションに追加されたときに、移行先パスが指定されていた場合に、ローカル ファイル システムにコピーされます。 アプリケーションがアンインストールされると、ファイルは、ファイル システムから削除されます。  
  
-   **証明書。** インストール中に、証明書は、ローカル コンピューターの他のユーザーの証明書ストアに追加されます。 アプリケーションがアンインストールされると、証明書は削除されません、開発者にはこの操作を実行するアプリケーションには処理後のスクリプトが含まれていない場合。 必要に応じて、証明書を手動で削除することができます。 詳細については、次を参照してください。 [BizTalk アプリケーションの設定およびその他のファイルを削除する方法](../core/how-to-remove-other-files-and-settings-for-a-biztalk-application.md)します。  
  
-   **仮想ディレクトリ。** アプリケーションがインストールされているコンピューターでインターネット インフォメーション サービス (IIS) が存在する場合、仮想ディレクトリが作成されます。 既に同じ名前を持つ仮想ディレクトリが存在する場合、指定したポートを使用する既存の仮想ディレクトリのセキュリティ設定のままし、アプリケーションの .msi ファイル内の仮想ディレクトリ リソースは既存の仮想ディレクトリに書き込まれます変更されていません。 これらのセキュリティ設定が十分なことを確認してください。 アプリケーションをインストールするときに、仮想ディレクトリがバインドされているアプリケーション プールが、ローカル コンピューター上に存在しない場合は、仮想ディレクトリが既定のアプリケーション プールにバインドされます。  
  
     アプリケーションがアンインストールされると、仮想ディレクトリとそのファイルは削除、インストールの前に、仮想ディレクトリが存在していたか、インストールが完了する仮想ディレクトリにファイルが追加された場合を除き、します。 この場合、アプリケーションの .msi ファイルからインストールされたファイルのみが削除されます。 仮想ディレクトリとアプリケーションのインストール後に追加されたファイルは削除されません。 空である場合にのみ、仮想ディレクトリが削除されます。  
  
-   **COM コンポーネントです。** インストール中に、COM コンポーネントは、コンポーネントがアプリケーションに追加されたときに、このオプションが指定されていた場合に、Windows レジストリに追加されます。 さらに、ファイルは、コンポーネントがアプリケーションに追加されたときに、コピー先の場所が指定されていた場合、ローカル ファイル システムにコピーされます。 アプリケーションがアンインストールされるには、開発者には、この機能を実行するアプリケーションに処理後のスクリプトが含まれていなければ、COM コンポーネントは、Windows レジストリまたはファイル システムから削除されません。 必要に応じて、Windows レジストリとファイル システムからの COM コンポーネントを手動で削除できます。 詳細については、次を参照してください。 [BizTalk アプリケーションの設定およびその他のファイルを削除する方法](../core/how-to-remove-other-files-and-settings-for-a-biztalk-application.md)します。  
  
-   **処理前および処理後のスクリプトです。** インストール中に、スクリプト ファイルは、ファイル、アプリケーションに追加されたときに、移行先パスが指定されていた場合に、ローカル ファイル システムにコピーされます。 処理前のスクリプトは、アプリケーションのインポートまたは削除する前に、アンインストールの後に実行します。 処理後のスクリプトは、アプリケーションのインポートまたは削除した後、インストールする前に実行します。 スクリプト ファイルは、アプリケーションのアンインストール時に、ファイル システムから削除されます。 ただし、前または処理後のスクリプトでは、ファイル システムにファイルが追加された場合は削除されません。 必要に応じて、ファイルを手動で削除することができます。 詳細については、次を参照してください。 [BizTalk アプリケーションの設定およびその他のファイルを削除する方法](../core/how-to-remove-other-files-and-settings-for-a-biztalk-application.md)します。  
  
-   **BAM アイテム。** インストール中に、BAM アイテム ファイルは、成果物は、アプリケーションに追加されたときに指定された宛先パスにコピーされます。 アプリケーションがインストールされている場合、BAM アイテムは展開されません。 アプリケーションがアンインストールされると、BAM アイテム ファイルは削除されます。
