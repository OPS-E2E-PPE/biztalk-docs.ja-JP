## <a name="a"></a>A  
 **アセンブラー**  
 A [!INCLUDE[btsCoName](../../includes/btsconame-md.md)] [!INCLUDE[btsBizTalkServerNoVersion](../../includes/btsbiztalkservernoversion-md.md)]送信パイプライン処理のアセンブル ステージ中に呼び出される送信パイプライン コンポーネントです。 通常はアセンブラーを使用すると、一部のフラット ファイル形式を XML から送信メッセージのシリアル化のジョブ。  
  
 **アセンブリ**  
 主なビルド ブロック、 [!INCLUDE[btsCoName](../../includes/btsconame-md.md)] [!INCLUDE[btsDotNetFramework](../../includes/btsdotnetframework-md.md)]アプリケーションです。 再利用、バージョン管理、セキュリティ、および展開の基本単位です。 プログラマが単一のダイナミック リンク ライブラリ (DLL) または実行可能 (EXE) に表示されるファイルのコレクションです。  
  
 **アセンブリ キャッシュ**  
 アセンブリのサイド バイ サイドの記憶域に使用するマシン全体のコード キャッシュ。 キャッシュに 2 つの部分があります。 グローバル アセンブリ キャッシュには、コンピューターに多数のアプリケーション間で共有できる明示的にインストールされているアセンブリが含まれています。 インターネットからダウンロードされたコードまたはイントラネット サイト、分離コードが 1 つのアプリケーションの代わりにダウンロードされるように、ダウンロードをトリガーしたアプリケーションをダウンロード キャッシュに格納またはページでは他のアプリケーションには影響しません。  
  
## <a name="b"></a>B  
 **銀行識別コード (BIC)**  
 SWIFT で定義されている、金融機関を識別するために使用するコードです。  
  
 **ビジネス ルール作成ツール**  
 ポリシーを作成するための GUI ツール。  
  
 **ビジネス ルール エンジン (BRE)**  
 ファクトに対してルールを評価し、評価結果に基づいてアクションを開始する実行時推論エンジン。  
  
## <a name="c"></a>C  
 **条件付き規則**  
 SWIFT メッセージの種類のフィールド間の関係を指定するルール。 SWIFT 標準リリース ガイドでは、条件付き規則が定義されています。

 **エラー コード**  
 指定されたメッセージの種類の規則の特定の違反を示す 2 桁の数字が続く文字から成るコードです。  
  
 **拡張マークアップ言語 (XML)**  
 W3C (World Wide Web Consortium) によって開発された仕様。標準 HTML の機能にとらわれずに独自のタグを作成できます。 定義済みのタグのみを使用しますが、ページ内の要素を記述する HTML、XML は、ページの開発者によって定義されるタグを使用できます。 特定のアプリケーションの製品や、期限を時間など、ほぼすべてのデータ項目のタグを使用できます。 そのため、Web ページがデータベース レコードとして機能するようになります。  
  
 **Extensible Stylesheet Language (XSL)**  
 拡張マークアップ言語 (XML) ドキュメントのスタイル シートの書式。 XSL を使用して、ハイパー テキスト マークアップ言語 (HTML) の表示を定義する、カスケード スタイル シート (CSS) を使用する方法と同じで、XML の表示を定義します。 [!INCLUDE[btsBizTalkServerNoVersion](../../includes/btsbiztalkservernoversion-md.md)]2 つの仕様間の変換言語として XSL を使用します。

 異なる仕様間のレコードとフィールドの対応関係を定義している XML ファイル。 マップには、拡張スタイル シート言語 (XSL) が含まれるスタイル シートが[!INCLUDE[btsBizTalkServerNoVersion](../../includes/btsbiztalkservernoversion-md.md)]を使用して、マップに記述された変換を実行します。 BizTalk マッパーでマップを作成します。  
  
 **メッセージの種類**  
 「支払に対して受信」など、SWIFT、標準リリース ガイドで定義されているメッセージ形式のいずれか。 メッセージの種類は、多くの場合、"MT"の後に、3 桁のコードで示されます。

## <a name="s"></a>S  
 **スキーマ**  
 XML ファイルの構造を定義したもの。 スキーマには、レコードと、構造内のフィールドに関連するプロパティ情報が含まれています。  
  
 **世界中銀行間通信 (SWIFT) の社会**  
 世界中銀行間通信の society です。 メッセージング サービス銀行、broker ディーラー、投資マネージャー、および支払い、財務、証券、および取引の市場インフラストラクチャを提供する組織です。 SWIFT は、世界中のデータの処理を共有し通信リンク、国際対応の財務トランザクションの共通言語を作成します。  
  
 **仕様**  
 [!INCLUDE[btsBizTalkServerNoVersion](../../includes/btsbiztalkservernoversion-md.md)] 固有の XML スキーマ。 仕様では、BizTalk エディターで作成され、またはフラット ファイル (区切り記号付き、位置指定、または区切り記号付きおよび位置指定) (SWIFT、EDIFACT、x12、および XML) などの業界標準に基づくことができます。 BizTalk マッパーは、送信元仕様と送信先仕様として公開された仕様を使用してマップを作成します。  
  
 **厳密な名前**  
 アセンブリの id で構成される名前-その単純テキスト名、バージョン番号、およびカルチャ情報 (存在する場合): 公開キーと、アセンブリに対して生成されたデジタル署名によって強化します。 アセンブリ マニフェストにアセンブリの実装を構成するすべてのファイルのファイル ハッシュが含まれているために、1 つのファイルだけがアセンブリ マニフェストを含むアセンブリに経由でデジタル署名の生成に十分です。 同じ厳密な名前でアセンブリが同一であるが必要です。  
  
 **ストレート処理 (STP)**  
 手動による介入する必要はありません、複数のステップ経由でメッセージの自動処理します。 通常、処理パスに SWIFT からの結果の金融機関; 内部システム トランザクションの投稿に金融機関でのメッセージの受信から適用または、外部命令または SWIFT またはその他の財務インフラストラクチャ; を介して結果メッセージの送信に金融機関にアクションの受信または、関連する SWIFT またはその他の財務インフラストラクチャを経由してメッセージを転送する 1 つまたは複数の内部の金融機関システムからします。  
  
 **SWIFT 標準リリース ガイド (SRG)**  
 FIN メッセージのセットを更新されたと提案された仕様を提供する SWIFT パブリケーション。 これは、すべてのメッセージ型、有効な値は、および各フィールド、各メッセージに適用されるネットワーク ルールと任意の使用に関する規則または共通プラクティス形式のフィールドとレイアウトを定義するドキュメントの複数のボリューム セットです。 この CD パブリケーションは SWIFT からのサブスクリプションのサービスで使用できます。  
  
 **システムは、メッセージ トレーラー (SYS) の開始**  
 メッセージに SWIFT ネットワークにより追加されたトレーラーは、FIN サービスにメッセージが生成されたことを示す、金融機関に送信されます。 例としては、ブロードキャスト、ユーザーの要求、およびレポートへの応答です。
