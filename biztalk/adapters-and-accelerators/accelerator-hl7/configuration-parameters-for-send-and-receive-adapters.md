# <a name="configuration-parameters-for-send-and-receive-adapters"></a>構成パラメーターを送信および受信アダプター
このセクションでは、構成パラメーターを送信し、受信アダプターの最小限の下位レイヤー プロトコル (MLLP) を提供します。 これらのパラメーターが 2 つの型に分類されます。 文字をブロックし、ネットワーク接続のパラメーター。  

 ブロックの文字設定に設定 MLLP トランスポートの種類を使用して送信ポートの MLLP トランスポートのプロパティ。  

 パラメーターを設定、ネットワーク接続 MLLP トランスポートのプロパティのいずれかの送信ポートまたは受信場所の MLLP トランスポートの種類を使用することができます。 これを行うに、BizTalk Server 管理コンソールを開くいずれかに該当する場合、送信ポートまたは受信場所のフォルダーに移動しますを右クリックして、特定の送信ポートまたは受信場所 をクリックして**プロパティ**順にクリックします**構成**します。  

## <a name="block-characters"></a>文字のブロック  
 これらのパラメーターは、特殊文字を HL7 メッセージの受信または送信 MLLP アダプター経由で囲む必要があります。 これらの文字は、次の形式でブロックを形成: \<SB\>*DDD*\<EB\>\<CR\>ここで、 *DDD*意味メッセージのデータでは、 \<SB\>開始ブロックの文字は、 \<EB\> end ブロックの文字と\<CR\>キャリッジ リターンが。  

|パラメーター|新しく使用する機能|  
|---------------|---------|  
|**\<CR\>キャリッジ リターン**|キャリッジ リターン (末尾バイトの後に 2 番目のバイト ラッパー) を使用する (16 進数形式) のバイト値。 任意。|  
|**\<EB\> End ブロックの文字**|末尾バイト (メッセージ トレーラー ラッパー) を使用するバイト値。 ASCII \<FS\>、たとえば、 \<1 c\>します。|  
|**\<SB\>開始ブロックの文字**|開始バイト (メッセージ ヘッダーのラッパー) を使用するバイト値。 ASCII \<VT\>、たとえば、 \<0b\>します。|  

## <a name="deliverymode"></a>DeliveryMode  
 インスタンス ファイルがシーケンス、または (順不同の順序) で順序どおりに配信されるかどうかは、コントロールへの配信モード パラメーターを使用します。 各受信場所がファイルのシーケンスのインスタンス、独自の配信します。  

|プロパティ|目的|  
|--------------|----------------|  
|順次配送|**MLLP 受信**:<br /><br /> 設定、**順次配送**プロパティを**TRUE**、指定した順序でメッセージを処理する必要があるかを指定します。 順次配送は、パートナーのビジネス プロセスの指定どおりに、メッセージをコンボイとして処理する必要がある場合に重要です。<br /><br /> 設定した場合、**順次配送**プロパティを**FALSE** (既定値) の場合は、ポートで順次配送が適用されません。|
